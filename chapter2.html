2주차

- 인라인 스크립트
  HTML 태그 안에 직접 작성하는 자바스크립트
  팝업창을 열고 닫거나, 알림 메시지를 표시할 때처럼 간단한 명령을 처리할 경우 사용

- 내부 스크립트
  <script>/</script> 사이에 실행할 스크립트 소스 작성
  스크립트 태그는 웹문서 모든 곳에 위치할 수 있고, 그 위치에서 바로 실행됨
  한 문서 안에 여러 개의 스크립트 태그 가능
  내부 스크립트는 주로 </body> 앞에 사용

- 인라인, 내부 스크립트의 단점
  같은 소스를 여러 웹문서에서 반복 사용하면 비효율적이고, 수정할 때도 모든 곳을 찾아서 고쳐야 함
  따라서 외부 스크립트 파일을 저장하여 링크하는 것이 좋음 (모듈화)

- 외부 스크립트 연결하기
  1. 확장자: .js
  2. HTML 문서에서 스크립트 태그를 사용해 외부 스크립트 연결
  3. <script src="스크립트 파일경로"></script>
  4. 외부 스크립트 파일은 스크립트 태그 없이 소스만 작성
  5. 보통 js 폴더를 만들어 저장하는 것이 좋음

- 연결 예시
  a\\b\\c.html 에서 a\\b\\c\\d.js 연결
  <body>
    <p>연결하는 방법</p>
    <script src="c/d.js"></script>
  </body>

- 자주 쓰는 메서드 및 함수
  document.write
  alert
  confirm
  console.log

- 입출력 방법
  *경고창 alert(): alert("안녕하세요")
  *확인창 confirm(): result = confirm("종료하시겠습니까?") (true 또는 false 반환)
  *프롬프트 창(prompt()): result = prompt("이름을 입력하세요 :") (문자열 반환, 취소시 null)
  *콘솔 로그(console.log()): 콘솔 창에 결과 출력

- 변수 사용 규칙
  1. 숫자로 시작 불가, 공백 금지
  2. 대소문자 구분, 보통 소문자 사용
  3. 두 단어 이상 변수는 언더바(_) 또는 카멜케이스 사용
  4. 예약어 사용 불가
  5. 무의미한 변수명 사용 지양

- 변수 선언
  let a = 0; // 재선언 불가, 재할당 가능
  const a = 0; // 재선언 및 재할당 불가
  var a = 0; // 재선언 및 재할당 가능, 호이스팅 있음

- 자료형
  1. 원시형
    - 숫자 (number)
    - 문자열 (string)
    - 논리형 (boolean, true/false)
    - NaN (Not a Number)
    - undefined
    - null
  2. 객체형 (배열, 객체 등)

- 자료 변환
  *숫자로 변환: Number(), parseInt(), parseFloat()
  *문자열로 변환: toString()
  *논리형 변환: Boolean()
